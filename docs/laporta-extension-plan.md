# archilang-laporta 拡張計画 (v0.1)

> ラポルタ(株式会社ラポルタ・世田谷区給田5-12-12)が VW2025 Architect で受注する建設・内装案件向けに ArchiLang を拡張する。
> 目的: **自然言語 → ArchiLang YAML → VW2025 平面図 + 見積JSON** を一気通貫で生成。

## 1. ArchiLang upstream の流用範囲

| Upstream機能 | 流用 | 備考 |
|---|---|---|
| YAML パーサ + Resolver (mm化) | ✅ そのまま | `parser.ts` / `solve.ts` |
| Validator (12種ルール) | ✅ そのまま | LLM フィードバックループの教師信号 |
| inspect (部屋グラフ + 隣接 + 占有グリッド) | ✅ そのまま | AI再生成の現状把握用 |
| 910mm shaku モジュール | ✅ そのまま | ラポルタ案件 (木造軸組住宅 + 内装) の8割で十分 |
| SVG/HTML レンダラ | ✅ そのまま (プレビュー用) | 暫定図出力 |
| EquipmentType (kitchen_counter / unit_bath / toilet / washbasin / washing_machine / refrigerator) | ✅ ベース | ラポルタ建具スタイル追加で拡張 |

## 2. ラポルタ拡張ポイント

### 2-1. cost-master 連動 (`src/laporta/cost-master.ts`)
- vw-plugin 側 `/Users/koki/vw-plugin/src/resources/cost-master.json` (v2.7.0 / 947品目) を読み込み
- ArchiLang `room.type` (LDK / 寝室 / 浴室 / 和室 / トイレ / 洗面脱衣 / キッチン / 子供部屋 / 主寝室) → cost-master room タグ → 関連品目リスト
- ArchiLang `equipment.type` (kitchen_counter / unit_bath / toilet / washbasin / washing_machine / refrigerator) → cost-master ID 直マッピング
  - kitchen_counter → WW-009 (造作キッチンカウンター ¥350,000)
  - unit_bath → 既存品目要確認、なければ別途追加
  - toilet → fixtures category 内
  - 等
- ArchiLang `walls[].type` (external/internal) + `thickness` → cost-master ID
- ArchiLang `openings[].style` (引違い窓 / 片開き / 引き戸) → fixtures category

### 2-2. 見積エミッタ (`src/laporta/estimate-emitter.ts`)
- 解決済み (resolved) Archilang モデル → 見積JSON出力
- 部屋面積 (㎡) × 関連品目 (例: クロス IN-001 ¥1,500/㎡ × LDK 28㎡)
- 開口部数 × 建具品目
- 設備数 × 設備品目
- 出力フォーマット (vw-plugin 互換):
```json
{
  "version": "2.7.0",
  "categories": [
    {"id": "interior", "items": [{"code":"IN-001", "name":"...", "qty":28, "unit":"㎡", "unitPrice":1500, "amount":42000}]}
  ],
  "subtotal": ...,
  "tax": ...,
  "total": ...
}
```

### 2-3. VW Marionette/Python 出力 (`src/laporta/vw-marionette-emitter.ts`)
- 解決済み Archilang モデル → VW Vectorscript Python スクリプト (.py)
- ラポルタテンプレ前提:
  - Layer: `平面-1F` (Story-bound, 1F=2700mm)
  - Class: `建築-壁-既存` / `建築-壁-新設` / `建築-壁-解体` / `建具-木製` / `建具-アルミ` / `窓-既存` / `窓-新設`
- API使用: `vs.AddWall()` / `vs.CreateDoor()` / `vs.CreateWindow()` / `vs.Move3DObj()`
- 出力例:
```python
# Generated by archilang-laporta v0.1
import vs
vs.NameLayer('平面-1F')
vs.NameClass('建築-壁-新設')
vs.AddWall((0, 0), (3000, 0))  # LDK南壁
# ...
```

### 2-4. ラポルタ実案件サンプル (`samples/laporta-*.yaml`)
- `laporta-30sqm-renovation.yaml`: 30㎡ 1LDK 内装リノベ (世田谷区アパート想定)
- `laporta-shibuya-winebar.yaml`: 渋谷ワインバー (Margaret Field 設計、店舗内装)
- `laporta-minamiaoyama-3f.yaml`: 南青山3丁目 オフィス (簡略版)

### 2-5. CLI コマンド拡張 (`src/main.ts`)
- `node dist/main.js to-vw <yaml> [out.py]` → VW Marionette Python 出力
- `node dist/main.js estimate <yaml> [out.json]` → 見積JSON出力
- `node dist/main.js full <yaml>` → SVG + Python + 見積 をまとめて出力

## 7. リスクと対応

| リスク | 対応 |
|---|---|
| cost-master の room タグが ArchiLang room.type と完全一致しない | エイリアステーブル `LDK→[リビング, ダイニング, キッチン]` を作る |
| VW Python API のシグネチャを間違える | vw-plugin 側の既存 import-vw コマンドを参考にする / 失敗時は手動補正 |
| LLM が ArchiLang 文法を完全に守らない | Validator エラーを LLM にフィードバックして自動修正ループ (max 3回) |
| 斜め壁/曲面が必要な案件 | Phase 1 では非対応。「ハイブリッド運用」として VW で人間が編集 |
